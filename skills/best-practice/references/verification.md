# best-practice — Verification Reference

Citation 검증 절차, 출처 독립성, 도메인별 시의성 기준.

## Citation 검증 절차

출처 보고 전 모든 인용을 검증한다. 검증 실패 항목은 출처에서 제외하거나 ⚠️ 태그.

### 단계별

1. **링크 살아있음 확인**
   - 모든 인용 URL을 WebFetch로 fetch
   - 200 OK 확인. 404/410/302→다른 페이지면 ⚠️ 또는 제외

2. **인용 내용이 실제 페이지에 있는가**
   - 인용한 문장의 핵심 키워드가 페이지에 등장하는지 확인
   - 환각 방지: "X가 Y를 했다"고 적었는데 그 페이지에 Y 언급이 없으면 인용 잘못된 것

3. **1차 출처 추적 (출처 독립성)**
   - 인용한 출처가 또 다른 출처를 인용한 거면 1차까지 추적
   - 예: "AWS 블로그가 K8s 공식 문서를 인용" → K8s 공식 문서를 1차 출처로 표시
   - 모든 출처가 결국 같은 1차 소스로 수렴하면 다양성 기준 미달로 판단

### 검증 결과 표기

```
## 출처

1. [How we manage 1000+ ArgoCD apps](https://...) - 블로그 ✅
   - 검증: 200 OK, 핵심 인용 확인됨
2. [AWS EKS 블로그](https://...) - 클라우드 공식 ⚠️ 1차 출처는 K8s 공식 문서
3. [Outdated link](https://...) - ❌ 404, 출처에서 제외
```

## 출처 독립성

### 다양성 vs 독립성

지금 Quality Criteria의 "출처 다양성"은 *카테고리* 기준이라 헛점이 있다.
형식적 다양성(공식+블로그+커뮤니티)을 만족해도 모두 같은 1차 소스를 인용하면
실질적으로는 **단일 소스 의존**(에코 챔버).

### 검증 규칙

- **같은 1차 소스를 인용하는 출처는 1개로 카운트**
- A가 B를 인용하고 B가 1차 출처면 → B만 인용에 표시
- 의견이 갈리는 주제는 **반드시 다른 입장의 출처 1개 이상** (반대편 입장)
- 단일 회사/조직의 시각만 있으면 ⚠️ "단일 관점" 경고

### 출력 시 표기

```
## 출처 다양성 평가

표면적: 5개 (공식 1, 블로그 2, 커뮤니티 2) ✅
실질적: 3개 독립 1차 소스
- K8s 공식 문서 (출처 1, 2가 인용)
- ArgoCD GitHub Discussion (출처 3)
- Reddit r/kubernetes 토론 (출처 4, 5는 같은 스레드)

⚠️ 단일 관점 경고: 모든 의견이 "AppSet 사용" 권장. 반대 입장 미발견
```

## 도메인별 시의성 기준

DevOps 분야는 변화 속도가 분야마다 다르다. 일률적인 "2년" 기준은 부정확.

### 도메인별 임계값

| 도메인 | "최신" 기준 | "오래됨" 경고 시점 |
|--------|------------|------------------|
| K8s API/CRD/Operator | 1년 | 1년+ |
| Cloud Provider 기능 (EKS, GKE, AKS) | 1년 | 1년+ |
| GitOps (ArgoCD/Flux) | 1년 | 1.5년+ |
| Service Mesh (Istio/Linkerd) | 1년 | 1.5년+ |
| Helm 패턴 | 2년 | 3년+ |
| 블록체인 RPC (EVM/Cosmos) | 6개월 | 1년+ |
| SRE 원칙/패턴 | 3년 | 5년+ |
| 보안 베스트프랙티스 | 1년 | 2년+ |
| Terraform 패턴 | 2년 | 3년+ |

### 적용 방식

1. 주제의 도메인을 분류 (1단계에서 이미 결정)
2. 출처의 발행일을 그 도메인 기준으로 평가
3. "Last updated" 또는 "Last verified" 날짜가 글에 있으면 그것 우선

### 출처 표기 예시

```
1. [How we manage 1000+ ArgoCD apps](https://...) - 블로그
   - 발행: 2023-04 ⚠️ 1.5년 전 (GitOps 도메인 경고 시점 도달)
   - 핵심 패턴은 여전히 유효, ArgoCD 2.7 → 2.13 사이 일부 변경 가능

2. [SRE Workbook Ch.4](https://google.com/...) - 공식 문서
   - 발행: 2018 (SRE 원칙 도메인 — 5년+ 기준 미해당, 여전히 유효)
```
