| 장점 | ... | ... | ... |
| 단점 | ... | ... | ... |

| GitHub Stars | ... | ... | ... |

## 업계 동향

## 학습 저장 프롬프트 (의미 있는 결과 시)

리서치 결과가 다음 중 하나에 해당되면, 출력 끝에 학습 저장을 **명시적으로 권유**한다:

- 명확한 결론에 도달함 (단순 정보 나열이 아닌 의사결정 근거 포함)
- 비자명한 인사이트 발견 (사용자가 몰랐을 가능성이 큰 패턴/한계)
- 사용자 환경에 직접 적용 가능한 권장 사항 도출
- 향후 비슷한 결정 시 재참조 가치가 있음

### 권유 메시지 템플릿

```
---

💡 이 리서치는 향후 비슷한 결정에 재활용 가치가 있어 보입니다.

학습 기록으로 저장하려면 `/compound completed best-practice <주제 요약>`을 실행하세요.
docs/learnings/<domain>.md 에 핵심 결론과 출처가 추가됩니다.

(휘발성 정보로 충분하면 무시해도 됩니다.)
```

### 권유하지 않는 경우

- 단순 정보 조회 ("X가 무엇인가" 정도)
- 결론이 모호하거나 출처 부족으로 ⚠️ 태그가 많은 경우
- 이미 사용자가 알고 있을 법한 자명한 내용

→ 노이즈 방지. 매번 권유하면 의미가 흐려진다.

## Quality Criteria

- **최소 출처 수**: 3개 이상 (가능하면 5개+)
- **출처 다양성 (표면적)**: 공식 문서 + 커뮤니티 + 블로그/사례 중 최소 2개 카테고리
- **출처 독립성 (실질적)**: 같은 1차 소스를 인용하는 출처는 1개로 카운트. 의견 갈리는 주제는 다른 입장 출처 1개 이상
- **시의성**: 도메인별 기준 적용 (verification.md 참조)
- **Citation 검증**: 모든 출처 URL fetch + 인용 내용 확인 후 보고
- **AI 콘텐츠 식별**: medium/dev.to 등 의심 신호 체크 (sources.md 참조)
- **근거 없는 주장 금지**: 모든 주장에 출처 필수
- **불확실성 명시**: 정보가 부족하거나 확인 불가한 부분은 "확인 필요"로 표기
