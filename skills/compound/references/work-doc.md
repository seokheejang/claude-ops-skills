# compound — Work Doc Reference

작업 문서(`docs/0X-*.md`) 작성 및 갱신 규칙. compound 워크플로 3단계 (A) 상세.

## 신규 작성 템플릿

기존 `0X-*.md` 패턴을 따른다. 신규 생성 시 아래 템플릿 사용:

```markdown
# <Task Title>

**날짜**: YYYY-MM-DD
**상태**: 진행중 | 일시중단 | 완료

## 배경

<왜 이 작업이 필요했는가. 동기, 요청 사항.>

## 변경 내용

<무엇을 했는가. 파일별, 기능별 정리.>

| 파일/디렉토리 | 변경 | 설명 |
|---------------|------|------|
| `path/to/file` | 신규/수정/삭제 | 간단 설명 |

## 결과

<성과, 테스트 결과, 관찰 사항.>

## 핸드오프

<paused: 다음 세션에서 바로 이어갈 수 있는 구체적 컨텍스트>
<completed: "→ CHANGELOG.md에 기록됨" 또는 생략>
```

## 기존 문서 업데이트 규칙

이미 `0X-*.md`가 존재하고 토픽이 일치하면:

- 날짜 헤더 (`### YYYY-MM-DD 추가 작업`) 로 새 섹션 추가
- **기존 내용은 절대 수정하지 않는다** — 추가만 (append-first)
- 같은 날 여러 작업이면 한 섹션 안에 시간순 bullet으로 묶음

## 번호 채번

- `docs/` + `docs/archive/` 양쪽에서 `[0-9][0-9]-` 접두사의 최대값 N을 찾는다
- 신규 문서는 `{N+1:02d}-<slug>.md`
- slug는 kebab-case, 짧고 검색 가능하게 (예: `15-mmdraw-skill.md`)

## 상태 전이

| 작성 시점 상태 | 다음 액션 |
|---------------|----------|
| 진행중 | 최상위(`docs/`)에 유지 |
| 일시중단 | 핸드오프 섹션 필수, 최상위 유지 |
| 완료 | `docs/archive/`로 이동 + CHANGELOG 링크 갱신 |
